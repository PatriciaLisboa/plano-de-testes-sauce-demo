# Plano de Testes

Este documento tem como objetivo descrever o escopo, a estratégia e os critérios utilizados para a execução dos testes realizados no Sauce Demo.

## Funcionalidades Testadas

As seguintes funcionalidades foram contempladas durante a execução dos testes:

* Login;
* Produtos;
* Carrinho de compras;
* Checkout;
* Logout.

---

## Estratégia dos Testes

Foram utilizados os seguintes tipos de testes durante a validação das funcionalidades:

* Testes positivos;
* Testes negativos;
* Testes funcionais;
* Testes exploratórios.

---

## Critério de Aprovação

Os testes serão considerados aprovados quando:

* Todos os casos de teste planejados forem executados com sucesso;
* As funcionalidades apresentarem o comportamento esperado durante a execução dos cenários definidos.

---

## Critério de Encerramento

A execução dos testes será considerada concluída quando:

* Todos os cenários planejados forem executados;
* A documentação do projeto estiver concluída;
* As evidências dos testes estiverem devidamente anexadas ao repositório.
